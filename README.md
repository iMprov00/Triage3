# TriagV3

Система триажа пациентов.

## Доступ администратора (seed)

- Логин: `admin`
- Пароль: `admin123`

## Запуск seed-данных

Чтобы создать или обновить администратора:

```bash
bundle exec rake db:seed
```

## Права пользователей

В системе есть 3 типа ролей (`job_positions.kind`):

- `admin` — администратор
- `doctor` — врач
- `other` — прочее

Ниже описано, что может каждая роль.

### 1) Работа с пациентом (карта пациента)

- **admin**
  - Может создавать/редактировать/удалять пациентов.
  - При создании/редактировании может выбрать исполнителем **любого пользователя** (`admin`, `doctor`, `other`).

- **doctor**
  - Может создавать/редактировать/удалять пациентов.
  - При создании/редактировании может выбрать исполнителем:
    - себя
    - любого пользователя с ролью `other`
  - Не может назначить исполнителем другого врача или администратора.

- **other**
  - Может создавать пациента только на себя (без выбора исполнителя).
  - **Не может удалять пациентов.**
  - Может редактировать карту пациента только в пределах общих ограничений приложения.

### 2) Активные шаги триажа (шаг 1/2/3)

- Любой авторизованный пользователь (`admin` / `doctor` / `other`) может вносить данные в **текущий активный шаг**.
- Это сделано специально для совместной работы смены.

### 3) Активные действия приоритета

- Любой авторизованный пользователь (`admin` / `doctor` / `other`) может выполнять действия на экране активного приоритета:
  - отмечать/снимать чекбоксы действий
  - фиксировать показатели (в сценарии red_arrest)
  - завершать действия при выполнении условий

### 4) Редактирование уже сохраненных шагов триажа

Маршруты редактирования уже сохраненных шагов (`/triage/edit_step/*`, preview/update):

- **admin / doctor** — могут редактировать.
- **other** — может редактировать **только своих пациентов** (где он исполнитель), иначе доступ запрещен.

### 5) Ограничение после завершения действий

- Если `actions_completed` уже установлен, редактирование шагов триажа недоступно для всех ролей (бизнес-ограничение процесса).
